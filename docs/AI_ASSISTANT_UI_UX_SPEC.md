# FanStudio AI 助手 UI/交互规范（V1）

## 1. 设计目标
- 友好：访客第一次使用就知道怎么问。
- 克制：不抢主内容注意力，不破坏现有前台气质。
- 可信：每条回答有来源、有图片，不“空口回答”。
- 一致：桌面与移动体验统一，学习成本低。

## 2. 视觉整合原则
- 复用现有站点色板、圆角、边框、阴影、字体体系。
- 不新增全局 reset，不覆盖现有组件样式。
- AI 组件使用独立 class 前缀：`fs-ai-*`。
- 交互动效轻量：开合/渐入/骨架屏为主。

## 3. 信息架构

### 3.1 桌面端
1. 右下角悬浮按钮（FAB）。
2. 点击后展开右侧对话面板（固定宽度，支持拖高）。
3. 面板分区：
- Header：标题、在线状态、关闭按钮。
- Quick Actions：快捷问题 chips。
- Message List：对话流。
- Composer：文本输入、语音按钮、发送按钮。

### 3.2 移动端
1. 右下角按钮触发底部抽屉。
2. 抽屉高度默认 80vh，可上滑全屏。
3. 输入框固定底部，消息区可滚动。

## 4. 消息卡片规范

### 4.1 用户消息
- 右对齐，简洁气泡，最多 85% 宽度。

### 4.2 助手消息
- 左对齐，包含以下可选模块（按顺序）：
1. `answer` 主回答文本
2. `images` 图片卡片横向列表
3. `citations` 引用来源列表
4. `relatedQuestions` 追问建议 chips

### 4.3 引用来源卡片
- 展示：来源标题 + 片段 + 类型标记（文章/作品/教程/关于）
- 行为：点击跳转到对应前台页面
- 数量：默认最多 3 条

### 4.4 图片卡片
- 展示：缩略图 + caption + 来源按钮
- 行为：点击放大预览；可跳来源页
- 数量：默认最多 3 张

## 5. 状态设计

### 5.1 会话状态
- `idle`：待输入
- `thinking`：AI 思考中
- `retrieving`：检索证据中
- `streaming`：流式输出中
- `error`：失败可重试

### 5.2 输入状态
- 空输入禁用发送按钮
- 发送中可中断（Stop）
- 网络失败时保留草稿

### 5.3 语音状态（V2）
- `listening`：正在录音
- `transcribing`：语音识别中
- `speaking`：语音播报中（可静音/打断）

## 6. 交互细节（关键体验）

### 6.1 首次使用
- 自动展示 3 个快捷问题：
  - 你擅长什么类型项目？
  - 有哪些代表作品？
  - 如何联系你合作？

### 6.2 回答可靠性反馈
- 有引用：显示“基于站内内容回答”。
- 无足够证据：显示“当前没有足够依据”，并推荐相关页面入口。

### 6.3 可用性细节
- Enter 发送，Shift+Enter 换行。
- 新消息到达时平滑滚动到底部。
- 长回答默认展开，超过阈值显示“收起/展开”。

## 7. 可访问性（A11y）
- 所有按钮有 `aria-label`。
- 对比度满足 WCAG AA。
- 语音功能可完全关闭，纯键盘可完成对话。
- 支持 `prefers-reduced-motion` 降低动画。

## 8. 组件清单（建议）
- `SiteAIAssistant`（总容器）
- `AssistantFab`
- `AssistantPanel`
- `AssistantHeader`
- `QuickQuestionChips`
- `MessageList`
- `UserBubble`
- `AssistantBubble`
- `CitationList`
- `ImageReferenceList`
- `Composer`
- `VoiceButton`（V2）

## 9. 与现有前台集成点
- 在前台布局客户端挂载全局助手组件（不侵入页面内容区）。
- 保持 `z-index` 低于系统级弹窗，高于普通卡片。
- 小屏避免遮挡底部导航，预留安全间距。

## 10. 错误与降级
- AI 未配置：显示“站长未启用助手”提示卡。
- 模型超时：显示“响应超时，请重试”并保留问题。
- 语音不可用：自动回退文本输入，不中断对话。

## 11. 文案语气建议
- 专业、简洁、礼貌。
- 避免夸张营销语和空泛赞美。
- 每次回答尽量给出“下一步建议”（继续追问或查看来源）。

## 12. 验收标准（UI/交互）
1. 与现有主题无冲突，不破坏原有页面布局。
2. 桌面与移动均可完整对话。
3. 每条 AI 回答可渲染 citations/images（若有）。
4. 加载、错误、空状态均有明确反馈。
5. 键盘操作和基础可访问性通过。

- 提供文案字典（可国际化）。
