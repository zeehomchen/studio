## 5. 前台设置不生效修复（force-dynamic）
- 问题：
  - 后台修改 favicon、网站名称等设置后，前台页面不生效，仍显示构建时的旧值。
  - 后台页面正常，因为后台是动态渲染（`ƒ Dynamic`）。
- 原因：
  - 前台页面在 `npm run build` 时被静态预渲染（`○ Static`），HTML 中的 `<head>` 标签（favicon、title、description 等）在构建时固化，后续数据库变更不会反映到已生成的静态页面。
- 实施内容：
  - 在根布局 `app/layout.tsx` 和前台布局 `app/(frontend)/layout.tsx` 中添加 `export const dynamic = "force-dynamic"`，强制每次请求都从数据库读取最新设置。
  - 修复后构建输出从 `○ Static` 变为 `ƒ Dynamic`。
- 变更文件：
  - `/Users/fan/code/fanstudio/src/app/layout.tsx`
  - `/Users/fan/code/fanstudio/src/app/(frontend)/layout.tsx`

## 6. BlockNote SlashMenu 类型错误修复
- 问题：
  - 生产构建（`npm run build`）时 TypeScript 报错：`Property 'key' does not exist on type 'DefaultReactSuggestionItem'`。
  - 本地开发模式不报错，因为 `next dev` 不做完整类型检查。
- 原因：
  - BlockNote 库将 `DefaultReactSuggestionItem` 定义为 `Omit<DefaultSuggestionItem, "key">`，类型层面移除了 `key` 属性，但运行时对象通过展开操作符实际包含 `key`。
- 实施内容：
  - 对 `item` 添加类型断言 `(item as unknown as { key: string }).key`，绕过类型限制。
- 变更文件：
  - `/Users/fan/code/fanstudio/src/components/admin/BlockNoteEditor.tsx`
